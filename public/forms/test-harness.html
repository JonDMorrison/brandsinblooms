🧪 BloomSuite Embed - Accessibility Test Harness

This page tests the accessibility implementation of modal and slide-in display modes. Use this for manual testing with keyboard navigation and screen readers.

📋 How to Run Tests Locally

  1. Serve this directory: npx serve public/forms or python -m http.server 8000
  2. Open http://localhost:8000/test-harness.html (or port 5000 for serve)
  3. Replace YOUR_EMBED_KEY_HERE with a valid 32-char embed key
  4. Run through each checklist item below
TEST 1

Inline Embed (Default)

Basic inline form rendering. No accessibility wrappers needed - just standard form controls.

TEST 2

Modal (Button Trigger)

Modal opens via button click. Tests focus trap, ESC handling, ARIA attributes, and focus restoration.

✅ Modal Accessibility Checklist

  • Click button → modal opens
  • Focus moves into modal automatically (close button or first field)
  • Tab cycles through modal elements only (focus trapped)
  • Shift+Tab cycles backwards within modal
  • Escape closes modal
  • Click overlay (outside modal) closes modal
  • Focus returns to the button that opened it
  • Background cannot be scrolled while modal is open
  • Screen reader announces "Form dialog opened"
TEST 3

Modal (Click Selector Trigger)

Modal opens when any element matching .cta-button is clicked.

✅ Click Trigger Checklist

  • Click CTA button → modal opens
  • Focus returns to CTA button on close
TEST 4

Host Page Conflict Test

Tests that BloomSuite modal ESC handler doesn't interfere with host page modals and vice versa.

✅ Isolation Checklist

  • Open host modal → ESC closes host modal only
  • Open BloomSuite modal → ESC closes BloomSuite modal only
  • BloomSuite modal z-index appears above host modal
TEST 5

Slide-In Panel (Button Trigger)

Panel slides from right edge. Same accessibility requirements as modal.

✅ Slide-In Accessibility Checklist

  • Click button → panel slides in from right
  • Focus moves into panel automatically
  • Tab cycles through panel elements only
  • Escape closes panel
  • Click overlay closes panel
  • Focus returns to trigger button on close
  • Title reads "Request a Quote" in header
TEST 6

Slide-In (Scroll Trigger at 25%)

Scroll down past 25% of the page height to trigger the slide-in panel.

✅ Scroll Trigger Checklist

  • Scroll past 25% → slide-in appears
  • Trigger fires only once per page load
  • Focus moves to panel when it opens
↓ Scroll down to test scroll trigger (25% point) ↓
KEYBOARD TESTING

Keyboard Navigation Summary

✅ Required Keyboard Behaviors

  • Tab - Move forward through focusable elements
  • Shift+Tab - Move backward through focusable elements
  • Enter - Activate buttons and submit forms
  • Space - Toggle checkboxes, activate buttons
  • Escape - Close modal/slide-in (only when focused inside)
SCREEN READER

Screen Reader Testing

Test with VoiceOver (Mac), NVDA (Windows), or TalkBack (Android).

✅ Screen Reader Behaviors

  • Modal opening announced: "Form dialog opened. Press Escape to close."
  • Modal closing announced: "Form dialog closed."
  • No duplicate announcements (debounced)
  • Dialog title is read when focus enters
  • Close button announces as "Close form, button"
iOS SAFARI

Mobile Safari Scroll Lock

On iOS Safari, test that background scroll is completely locked when modal/slide-in is open.

✅ iOS Safari Behaviors

  • Open modal → background cannot scroll
  • No "bounce" effect at top/bottom of modal
  • Close modal → scroll position restored exactly
  • Address bar behavior unchanged after close
