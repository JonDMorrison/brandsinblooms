## Objective

---

## Scope

### Follow-Up Chips: `src/components/bloom/BloomFollowUpChips.tsx`

- Renders 2-4 `JoyChip` variant="outlined" color="primary" size="sm" in a horizontal `Stack` with `flexWrap: "wrap"` and `gap: 1`
- Chips generated by the LLM (parsed from `<follow_ups>["chip1", "chip2", "chip3"]</follow_ups>` in response)
- Stored in `bloom_messages.follow_up_chips` as JSONB array
- Click handler sends the chip text as the next user message (calls `sendMessage()` from `BloomContext`)
- Chips animate in on message completion with staggered `framer-motion` delays (50ms between each)
- Chips disappear once the user sends the next message (only shown on the latest assistant message)

### Message Actions: `src/components/bloom/BloomMessageActions.tsx`

Rendered on hover (desktop) or long-press (mobile) at the top-right corner of every message:

**User message actions:**
- Edit (Pencil icon) â€” makes the message text editable inline, resend on Enter
- Copy (Copy icon) â€” copies message text to clipboard

**Assistant message actions:**
- Copy (Copy icon) â€” copies the text/markdown content (not block data)
- Bookmark (Star icon, filled when bookmarked) â€” toggles `is_bookmarked` via `useBloomMessageMutations`
- Regenerate (RefreshCw icon) â€” re-sends the preceding user message for a fresh response
- Thumbs up / Thumbs down (ThumbsUp, ThumbsDown icons) â€” stores feedback in message metadata JSONB

Actions render as a `Stack` direction="row" of `JoyButton` size="icon" variant="ghost" with `JoyTooltip` on each. The bar uses `position: absolute` with `top: -8px, right: 0` relative to the message, appearing with a quick fade-in (100ms).

### Bookmark Section in Sidebar

Add a "Saved" section toggle in `BloomSidebar.tsx`:
- `JoyButton` variant="plain" at the top of sidebar: "Saved ({count})"
- When toggled, sidebar shows bookmarked messages across all conversations, grouped by date
- Each bookmark shows: conversation title, message preview, timestamp
- Click navigates to the conversation and scrolls to the bookmarked message

# BLOOM-M19: Conversation Compaction Engine

> **Copilot Thinking Effort:** High
> **Branch:** `feature/bloom-assist`
> **Phase:** 3 â€” Streaming & Core UX
> **Milestone:** 19 of 40

---

## Objective

Implement the 3-tier conversation compaction system from Document 02. When conversations exceed 12 messages, older messages are summarized to stay within the 25K token budget while preserving context. This ensures Bloom maintains conversation coherence even in very long sessions.

---

## Scope

### Compaction Logic in `supabase/functions/bloom-assist/compaction.ts`

**Tier 1: Full Context (messages 1-8)**
- Last 8 messages sent to the LLM in full. No compaction.

**Tier 2: Condensed Summary (messages 9-20)**
- Messages 1 through (total - 8) are summarized into a condensed summary paragraph
- Summary generated by gpt-4o-mini: "Summarize this conversation so far in 2-3 paragraphs, preserving: entity names, filter criteria used, decisions made, and any preferences expressed."
- Summary stored in `bloom_conversations.metadata.compaction_summary`
- Summary injected as a system message at the start of Layer 5 (conversation history)
- Original messages marked as `is_compacted = true` in `bloom_messages` but NOT deleted

**Tier 3: Aggressive Summary (messages 20+)**
- Only the compaction summary + last 6 messages sent to LLM
- Summary refreshed every 10 new messages
- Maximum summary length: 800 tokens

### Smart Context Recall

When the user references earlier context ("use the same filters", "like before", "what we discussed"):
1. Detect reference via keywords in the user's message
2. Search full message history (from Supabase, not from context) for matching tool executions
3. Inject the recalled tool parameters as a "Recalled Context" block in the current request
4. Continue normal processing with the recalled data available

### Trigger Points

Compaction runs automatically when:
- A new message is added and `message_count > 12` and no compaction summary exists
- `message_count > 20` and the summary is older than 10 messages
- The context builder detects the assembled context exceeds 20K tokens (emergency compaction)

---

## Acceptance Criteria

- [ ] Messages 1-(N-8) are summarized into a condensed paragraph
- [ ] Summary preserves entity names, filter criteria, decisions, and preferences
- [ ] Summary stored in `bloom_conversations.metadata.compaction_summary`
- [ ] Original messages NOT deleted â€” marked `is_compacted = true`
- [ ] Last 8 messages always sent in full
- [ ] Summary refreshes every 10 new messages in long conversations
- [ ] Smart context recall finds relevant earlier tool executions
- [ ] Emergency compaction triggers if context exceeds 20K tokens
- [ ] Total context stays within 25K budget after compaction

---

## What NOT To Do

- Do NOT delete compacted messages â€” they must remain for recall and audit
- Do NOT compact the last 8 messages â€” those are always sent in full
- Do NOT use gpt-4o for compaction summaries â€” use gpt-4o-mini (cheaper, fast enough)
- Do NOT generate test files or documentation
