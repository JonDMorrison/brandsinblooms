# v1.1.0 Editor And Authoring Experience

## Summary

The Form Builder editor is a full authoring surface, not a thin JSON editor. The repo contains a forms index, a multi-tab editor, autosave orchestration, drag-and-drop field authoring, templates, preview tooling, and publish validation.

Primary routes:

- `/crm/forms`
- `/crm/forms/:formId`

Route definitions live in `src/App.tsx`.

## Forms Index Surface

Primary page:

- `src/pages/crm/FormsPage.tsx`

Confirmed capabilities:

- create a form from scratch via `createStarterForm()` in `src/lib/formTemplates.ts`
- open a template picker via `src/components/forms/FormTemplatesDialog.tsx`
- duplicate an existing form
- archive and delete forms
- copy the public form URL when an `embed_key` exists
- filter and sort forms by status, recency, and submission count

Primary hook:

- `src/hooks/useForms.ts`

## Editor Shell

Primary page:

- `src/pages/crm/FormEditorPage.tsx`

Confirmed tabs:

- Build
- Design
- Audience
- Compliance
- Submissions
- Analytics

Confirmed editor behavior:

- uses `src/hooks/useFormEditor.ts` for hydration and autosave
- tracks unsaved changes and guards navigation
- supports inline form-name editing in the header
- provides publish validation before status changes
- opens preview and share flows from the editor shell

## Build Tab

Primary files:

- `src/components/forms/FormBuildTab.tsx`
- `src/components/forms/DraggableFieldList.tsx`
- `src/lib/forms/fieldRegistry.ts`
- `src/lib/forms/formFlow.ts`

Confirmed authoring features:

- drag-and-drop field ordering using `@hello-pangea/dnd`
- field palette and field creation from registry defaults
- supported field types: email, text, phone, select, checkbox, file, hidden, email consent, SMS consent
- single-instance protection for consent field types
- step-based authoring with a Multi-Step toggle
- per-field step assignment
- conditional visibility editor with rule builder UI
- CRM mapping-key authoring with suggestions and free-text support
- regex validation authoring, including custom `pattern_message`
- file upload constraint authoring:
  - `max_files`
  - `max_file_size_mb`
  - `allowed_mime_types`
- hidden field default value authoring

The conditional visibility editor is real and shipped. It is not only a type-level placeholder.

Primary files:

Confirmed capabilities:

- live preview with debounce

- background simulation
- reset behavior

Important note:

- the editor preview is a local authoring aid
- it is not the same code path as the static `public/forms/embed*.js` runtime

## Known Gaps

- There is no form version history or rollback system in the current editor surface.
- There is no first-class builder UI for advanced embed trigger attributes like `data-trigger`, `data-delay`, `data-scroll-depth`, or `data-click-selector`, even though the public runtime supports them.
- `double_opt_in` is visible in the editor but still marked Coming Soon and not enforced in the submission pipeline.
