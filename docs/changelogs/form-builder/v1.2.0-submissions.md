# v1.2.0 Submission Pipeline And Admin Review

## Summary

The Form Builder submission layer is end-to-end: public form loading, payload normalization, server-side sanitization, customer upsert, audit storage, realtime updates, export, and batch deletion.

This area is backed by both untagged January foundation work and later explicit `FB-002` and `FB-009` milestones.

## Public Submission Flow

### 1. Public Form Route

Primary route and page:

- `/f/:embedKey`
- `src/pages/PublicFormPage.tsx`

Confirmed responsibilities:

- load form configuration from `GET /functions/v1/get-form-config`
- render the public form via `src/components/forms/preview/FormPreviewRenderer.tsx`
- normalize outgoing payload keys using `mapping_key`
- inject hidden field defaults
- pass page URL, referrer, and user agent into the submission metadata

### 2. Submission Sanitization

Server-side shared helper:

- `supabase/functions/_shared/formVisibility.ts`

Confirmed behavior:

- hidden fields stay active and can inject default values
- invisible conditional fields are stripped from the final stored payload
- sanitized output is keyed by `mapping_key` first, then field fallback identifiers

Test coverage:

- `src/__tests__/formVisibility.test.ts`

### 3. Public Submit Endpoint

Primary file:

- `supabase/functions/submit-form/index.ts`

Confirmed responsibilities:

- validate `embed_key`
- load the published form and derive `tenant_id`
- perform atomic rate limiting through `upsert_rate_limit(...)`
- perform honeypot spam detection
- validate required fields, format rules, regex rules, and consent rules
- create or update `crm_customers`
- store immutable consent metadata in `form_submissions.metadata`
- finalize uploaded files for accepted submissions
- apply personas and tags from `audience_json`

Stored submission outcomes:

- `accepted`
- `rejected_invalid`
- `rejected_rate_limited`
- `rejected_spam`

## Milestone Mapping

- `FB-002`: confirmed via the initial paginated submissions RPC
- `FB-009`: confirmed via the redesigned search, sort, summary, and delete contract

## Known Gaps

- There is no partial-submission or drop-off capture pipeline.
- There is no submission replay feature.
- There is no support-side note/tag workflow directly on `form_submissions`.
- The developer test matrix hits live endpoints and is operational tooling, not automated release protection.
