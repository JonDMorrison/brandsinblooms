# v1.5.0 Analytics And Reporting

## Summary

Analytics is not a client-side afterthought. The repo shows server-side aggregation contracts for both the forms index and the analytics tab, with the frontend consuming typed RPC responses instead of pulling raw submission rows into the browser.

## Forms Index Stats

Primary migration:

- `supabase/migrations/20260403170000_fb002_form_builder_server_aggregation.sql`

Primary consumer:

- `src/hooks/useForms.ts`

Primary page:

- `src/pages/crm/FormsPage.tsx`

Confirmed `get_forms_with_stats(...)` outputs used in the UI:

- `total_submissions`
- `recent_submissions`
- `recent_accepted`
- `recent_rejected`
- `last_submission_at`

This eliminates per-form client-side submission counting on the forms index.

## Analytics Dashboard Contract

Initial RPC foundation:

- `supabase/migrations/20260403170000_fb002_form_builder_server_aggregation.sql`

Expanded contract:

- `supabase/migrations/20260405133000_fb010_analytics_dashboard.sql`

Primary consumer:

- `src/hooks/useForms.ts`

Primary UI:

- `src/components/forms/FormAnalyticsTab.tsx`

Confirmed `get_form_analytics(...)` capabilities:

- current and previous period summaries
- accepted and rejected submission counts
- invalid, rate-limited, and spam breakdowns
- acceptance and rejection rates
- daily accepted and rejected trend points
- top referrers
- field fill-rate analytics
- `p_days <= 0` support for all-time mode
- conversion section with a first-class unavailable state

## Frontend Analytics Surface

Primary types:

- `src/types/formBuilder.ts`

Confirmed UI features in `FormAnalyticsTab`:

- range presets
- summary stat cards
- trend comparison labels
- daily chart rendering
- rejection breakdown chart

- field fill-rate reporting

## Milestone Mapping

- `FB-002`: initial analytics RPC foundation
- `FB-010`: richer analytics response and dashboard expansion

## Known Gaps

- There is no true form-view tracking pipeline yet, so conversion rate is explicitly allowed to be unavailable.
- There is no field-level drop-off analysis because partial submissions are not stored.
- There is no geographic or device breakdown in the current contract.
- There is no custom comparison builder beyond the preset date windows supported by the current UI and RPC shape.
